# Tool Finder 页面体系映射

## 目标
把 `Tool Finder` 的内容与页面方法，映射成适合“中文内容创作者 / 运营人员 AI 工具导航站”的页面原型。

重点不是复制它的视觉，而是借用它的：
- 页面类型
- 内容结构
- 内链逻辑
- 转化路径

## 一、先说结论

`Tool Finder` 的核心不是工具目录，而是：
- 高意图列表页
- 有判断的工具页
- 强对比页
- 替代品页
- 明确商业转化位

映射到你的项目后，首期最重要的页面体系应该是：
- 首页
- 分类页
- 工具详情页
- 对比页
- 场景页 / Best-of 页
- 替代品页
- 方法论页

## 二、Tool Finder 的页面类型，对应你该做什么

### 1. Tool Finder 的 Best-of 页
它在做什么：
- 按人群、场景、系统、任务切分榜单
- 每一页都回答一个明确问题

你该怎么映射：
- 做成 `场景页 / Best-of 页`

你的页面例子：
- 最适合小红书内容策划的 AI 工具
- 最适合公众号改写的 AI 工具
- 最适合短视频脚本生成的 AI 工具
- 最适合内容运营团队的 AI 工具

你要学的结构：
- 这个场景的问题是什么
- 推荐标准是什么
- 推荐哪些工具
- 每个工具为什么入选
- 新手选谁，进阶选谁，团队选谁

不要学的点：
- 不要切太散
- 不要一开始就做几十种人群页

## 三、Tool Finder 的单工具页

它在做什么：
- 不只是介绍工具
- 而是明确说“谁该用、谁不该用、真实使用会遇到什么”

你该怎么映射：
- 做成 `工具详情页`

你的工具页必须有：
- 工具概述
- 适用场景
- 价格摘要
- 中文支持
- 中国区可访问性说明
- 优点
- 缺点
- 适合谁
- 不适合谁
- 替代品
- 编辑结论
- CTA

你要额外强化的字段：
- 中国区访问是否稳定
- 是否需要海外支付
- 是否适合中文内容生产
- 是否适合团队协作

这一步是你和普通导航站拉开差距的地方。

## 四、Tool Finder 的对比页

它在做什么：
- 给两个工具做 head-to-head 判断
- 不是列配置，而是帮助用户选型

你该怎么映射：
- 做成 `对比页`

你的页面例子：
- ChatGPT vs Claude
- Jasper vs Copy.ai
- CapCut vs Descript
- Notion AI vs ChatGPT

推荐结构：
- 适合谁选 A
- 适合谁选 B
- 中文支持差异
- 中国区可用性差异
- 价格差异
- 功能侧重点差异
- 最终推荐结论

Tool Finder 可学的点：
- 结论先行
- 强调具体人群和工作流

## 五、Tool Finder 的 Alternatives 页

它在做什么：
- 围绕一个工具建立替代品网络
- 抢高意图搜索词

你该怎么映射：
- 做成 `替代品页`

你的页面例子：
- ChatGPT alternatives for Chinese users
- Claude alternatives for content creators
- Jasper alternatives for Chinese marketers

适合你的原因：
- 这类词天然有选型意图
- 更接近商业点击
- 很适合加入“中国用户可用性”判断

推荐结构：
- 为什么有人想找替代品
- 最值得看的 3 到 5 个替代工具
- 谁适合从原工具切走
- 中国用户应优先看哪些选项

## 六、Tool Finder 的分类页

它在做什么：
- 给用户按大类浏览入口
- 但真正有价值的是分类下的推荐和筛选

你该怎么映射：
- 做成 `分类页`

你的分类例子：
- AI 写作与改写工具
- 社媒内容策划工具
- 短视频脚本与视频生成工具
- AI 设计与内容包装工具
- 内容工作台与知识整理工具

推荐结构：
- 分类说明
- 适用场景
- 工具列表
- 筛选器
- 编辑推荐
- 相关对比页
- FAQ

注意：
- 分类页不能只是一堆卡片
- 必须有编辑上下文和推荐逻辑

## 七、Tool Finder 的 About / Trust 层

它在做什么：
- 强调“真实测试”
- 强调“有经验的编辑”
- 强调“怎么赚钱、但不影响判断”

你该怎么映射：
- 做成 `Methodology / About / Disclosure`

你的页面必须说明：
- 为什么做这个站
- 收录标准是什么
- 怎么评估中文支持
- 怎么评估中国区可访问性
- 工具有没有实际试用
- 如何处理联盟链接和赞助

这不是可有可无的页面，而是信任基础设施。

## 八、Tool Finder 的 Deals 层

它在做什么：
- 把商业化前置成“帮你省钱”
- 不是单纯埋 affiliate link

你该怎么映射：
- 首期先不做独立 deals 首页
- 后期做“精选工具优惠 / 合作推荐位”

更适合你的演进路径：
- 第一阶段：详情页和对比页内放普通 CTA
- 第二阶段：给可合作工具加联盟链接
- 第三阶段：再考虑“精选优惠”专区

不要现在就学它把首页做成优惠站。

## 九、Tool Finder 的内容组织方式，你应该怎么抄

它真正厉害的不是页面数量，而是页面之间的关系。

你应该这样搭：

### 首页
链接到：
- 重点分类页
- 热门场景页
- 热门对比页
- 热门工具页

### 分类页
链接到：
- 工具详情页
- 场景页
- 对比页

### 工具详情页
链接到：
- 替代品页
- 对比页
- 场景页
- 同分类其他工具

### 对比页
链接到：
- 两个工具详情页
- 相关场景页
- 替代品页

### 场景页
链接到：
- 多个工具详情页
- 多个对比页

这套结构既利于 SEO，也利于点击转化。

## 十、映射后的首期页面原型清单

### 首页原型
模块建议：
- Hero：一句话说清定位
- 搜索框
- 热门场景入口
- 热门分类入口
- 热门对比页
- 编辑精选工具
- 邮件订阅 CTA

### 分类页原型
模块建议：
- 分类介绍
- 筛选器
- 工具列表
- 编辑推荐
- 相关场景页
- 常见问题

### 工具详情页原型
模块建议：
- 工具名称与一句话定位
- CTA
- 价格 / 中文支持 / 中国区可用性
- 核心功能
- 优缺点
- 适合谁 / 不适合谁
- 替代品
- 相关文章

### 对比页原型
模块建议：
- 开场结论
- 并排对比表
- 不同用户怎么选
- 价格对比
- 中国区使用差异
- CTA

### 场景页原型
模块建议：
- 场景问题定义
- 推荐标准
- 推荐工具清单
- 每个工具的场景理由
- 新手 / 进阶 / 团队建议

### 方法论页原型
模块建议：
- 站点定位
- 评估标准
- 收录标准
- 更新频率
- 商业披露

## 十一、你最该学习 Tool Finder 的三件事

### 1. 先回答“谁适合谁”
不是先列功能，而是先给判断。

### 2. 先做高意图页面
不是做“AI 工具大全”，而是做“这个场景下该选谁”。

### 3. 建立强信任层
不是匿名目录，而是有方法、有标准、有结论。

## 十二、你不该照搬 Tool Finder 的三件事

### 1. 不要做太宽
你必须保持赛道更窄。

### 2. 不要先做重 deals
你前期还没有足够信任基础。

### 3. 不要把中国本地化判断做弱
这是你的核心差异化，不是补充项。

## 十三、一句话落地结论

如果用一句话概括映射关系：

`Tool Finder 负责告诉你怎么做“高信任的软件选型站”，而你的项目要把这套方法用在“中文内容创作者的 AI 工具选择”上。`
